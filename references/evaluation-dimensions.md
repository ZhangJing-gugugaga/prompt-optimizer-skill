# 评估维度详解

## 维度一：完整性（Completeness）

### 定义
提示词是否包含完成任务所需的所有信息。

### 评估要素

#### 1. 角色设定
- **完整**：明确定义角色身份、专业背景、能力范围
- **基本完整**：定义角色身份，但缺少背景或能力说明
- **不完整**：角色定义模糊或缺失

#### 2. 任务描述
- **完整**：清晰描述任务目标、背景、约束、期望结果
- **基本完整**：描述任务目标，但缺少背景或约束
- **不完整**：任务描述模糊或缺失关键信息

#### 3. 执行步骤
- **完整**：提供详细的执行步骤，包含输入输出和验证标准
- **基本完整**：提供步骤，但缺少输入输出或验证标准
- **不完整**：步骤模糊或缺失

#### 4. 输出要求
- **完整**：明确指定格式、长度、风格、质量标准
- **基本完整**：指定部分要求，但有遗漏
- **不完整**：输出要求模糊或缺失

#### 5. 约束条件
- **完整**：明确列出所有限制和边界
- **基本完整**：列出主要约束，但有遗漏
- **不完整**：约束条件模糊或缺失

### 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 所有要素完整，无遗漏 |
| 4 | 大部分要素完整，有1-2个次要遗漏 |
| 3 | 核心要素完整，有明显遗漏 |
| 2 | 部分要素完整，有重大遗漏 |
| 1 | 要素严重缺失 |

## 维度二：清晰性（Clarity）

### 定义
提示词是否表述清晰，无歧义。

### 评估要素

#### 1. 术语使用
- **清晰**：术语定义明确，使用一致
- **基本清晰**：大部分术语明确，有1-2个模糊
- **不清晰**：术语定义模糊或使用不一致

#### 2. 逻辑连贯性
- **清晰**：逻辑清晰，步骤连贯
- **基本清晰**：逻辑基本清晰，有1-2处跳跃
- **不清晰**：逻辑混乱，步骤跳跃

#### 3. 歧义程度
- **清晰**：表述明确，无歧义
- **基本清晰**：大部分明确，有1-2处歧义
- **不清晰**：多处歧义，需要解释

#### 4. 指令明确性
- **清晰**：指令明确，可直接执行
- **基本清晰**：指令基本明确，有1-2处模糊
- **不清晰**：指令模糊，需要猜测

### 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 表述完全清晰，无歧义 |
| 4 | 表述基本清晰，有1-2处小模糊 |
| 3 | 表述一般，有歧义需要解释 |
| 2 | 表述模糊，多处歧义 |
| 1 | 表述混乱，难以理解 |

## 维度三：可执行性（Executability）

### 定义
AI是否能直接执行提示词，无需额外解释。

### 评估要素

#### 1. 步骤可操作性
- **可执行**：步骤明确，可直接操作
- **基本可执行**：步骤基本明确，有1-2处需要解释
- **不可执行**：步骤模糊，无法直接操作

#### 2. 输入输出定义
- **可执行**：输入输出明确，有具体示例
- **基本可执行**：输入输出基本明确，缺少示例
- **不可执行**：输入输出模糊或缺失

#### 3. 验证标准
- **可执行**：有明确的验证标准和方法
- **基本可执行**：有验证标准，但不够具体
- **不可执行**：缺少验证标准

#### 4. 错误处理
- **可执行**：有明确的错误处理机制
- **基本可执行**：有错误处理，但不够完善
- **不可执行**：缺少错误处理

### 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 完全可执行，无需额外解释 |
| 4 | 基本可执行，有1-2处需要小解释 |
| 3 | 部分可执行，有明显障碍 |
| 2 | 执行困难，需要大量补充 |
| 1 | 无法执行 |

## 维度四：结构性（Structure）

### 定义
提示词是否逻辑清晰，结构合理。

### 评估要素

#### 1. 层次结构
- **清晰**：有清晰的层次结构，主次分明
- **基本清晰**：有层次结构，但有小问题
- **不清晰**：层次混乱，主次不分

#### 2. 顺序合理性
- **清晰**：顺序合理，符合逻辑
- **基本清晰**：顺序基本合理，有1-2处可优化
- **不清晰**：顺序混乱，不符合逻辑

#### 3. 分组合理性
- **清晰**：分组合理，相关元素在一起
- **基本清晰**：分组基本合理，有1-2处可优化
- **不清晰**：分组混乱，相关元素分离

#### 4. 格式规范性
- **清晰**：格式规范，易于阅读
- **基本清晰**：格式基本规范，有小问题
- **不清晰**：格式混乱，难以阅读

### 评分标准

| 分数 | 标准 |
|------|------|
| 5 | 结构完全清晰，层次分明 |
| 4 | 结构基本清晰，有小问题 |
| 3 | 结构一般，有明显问题 |
| 2 | 结构混乱，难以跟随 |
| 1 | 无结构，杂乱无章 |

## 综合评分

### 计算方法
综合评分 = (完整性 + 清晰性 + 可执行性 + 结构性) / 4

### 等级划分

| 等级 | 分数范围 | 说明 |
|------|---------|------|
| A | 4.5-5.0 | 优秀：可直接使用 |
| B | 3.5-4.4 | 良好：需要小修改 |
| C | 2.5-3.4 | 一般：需要较大修改 |
| D | 1.5-2.4 | 较差：需要重写 |
| E | 1.0-1.4 | 很差：需要重新设计 |

## 使用建议

### 评分后处理
- **A级**：直接使用，无需修改
- **B级**：根据问题清单进行小修改
- **C级**：根据改进建议进行较大修改
- **D级**：根据问题清单重写
- **E级**：重新设计提示词结构

### 持续改进
1. 使用提示词执行任务
2. 收集执行结果反馈
3. 根据反馈优化提示词
4. 重复验证直到达到A级
