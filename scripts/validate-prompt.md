# 提示词验证脚本

## 验证维度

### 1. 完整性检查
**目标**：确保提示词覆盖所有必要信息

**检查项**：
- [ ] 角色设定是否完整
- [ ] 任务描述是否清晰
- [ ] 约束条件是否明确
- [ ] 输出要求是否具体
- [ ] 执行步骤是否完整

**评分标准**：
- 5分：覆盖所有关键要素
- 4分：覆盖大部分要素，有1-2个遗漏
- 3分：覆盖核心要素，有明显遗漏
- 2分：覆盖部分要素，有重大遗漏
- 1分：要素严重缺失

### 2. 清晰性检查
**目标**：确保提示词无歧义

**检查项**：
- [ ] 术语是否定义明确
- [ ] 逻辑是否连贯
- [ ] 是否存在歧义表述
- [ ] 指令是否明确

**评分标准**：
- 5分：表述清晰，无歧义
- 4分：表述基本清晰，有1-2处模糊
- 3分：表述一般，有歧义
- 2分：表述模糊，多处歧义
- 1分：表述混乱，难以理解

### 3. 可执行性检查
**目标**：确保AI能直接执行

**检查项**：
- [ ] 步骤是否可操作
- [ ] 是否有明确的输入输出
- [ ] 是否有验证标准
- [ ] 是否有错误处理

**评分标准**：
- 5分：完全可执行，无需额外解释
- 4分：基本可执行，有1-2处需要解释
- 3分：部分可执行，有明显障碍
- 2分：执行困难，需要大量补充
- 1分：无法执行

### 4. 结构性检查
**目标**：确保提示词逻辑清晰

**检查项**：
- [ ] 是否有清晰的层次结构
- [ ] 是否有合理的顺序
- [ ] 是否有必要的分组
- [ ] 是否有适当的格式

**评分标准**：
- 5分：结构清晰，层次分明
- 4分：结构基本清晰，有小问题
- 3分：结构一般，有明显问题
- 2分：结构混乱，难以跟随
- 1分：无结构，杂乱无章

## 验证流程

1. **自动检查**：使用上述维度进行评分
2. **问题识别**：列出每个维度的具体问题
3. **改进建议**：针对每个问题提供改进方案
4. **重新验证**：改进后重新评分

## 输出格式

```markdown
## 验证报告

### 总体评分
- 完整性：X/5
- 清晰性：X/5
- 可执行性：X/5
- 结构性：X/5
- **综合评分：X/5**

### 问题清单
1. [维度]：[具体问题]
2. [维度]：[具体问题]

### 改进建议
1. [问题]：[改进方案]
2. [问题]：[改进方案]

### 改进后版本
[改进后的提示词]
```
